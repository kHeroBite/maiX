# kTest_deploy — 배포

> **프로젝트별 배포 설정**: kTest_deploy_{project} 참조

## 배포 실행 방식

```yaml
핵심_원칙: 빌드 완료 즉시 배포 (빌드와 배포가 하나의 단위)
  - Phase 1에서 각 프로젝트별로 빌드 → 배포를 연속 실행
  - 다른 프로젝트의 빌드 완료를 기다리지 않음

병렬_실행:
  MarsGW: 빌드 완료 → 즉시 원격 서버 배포
  Mars: 빌드 완료 → 즉시 프로세스 교체 (종료 → 실행)
  Mars.Mobile: 빌드 완료 → 즉시 에뮬레이터/디바이스 설치
```

## 배포 유형

```yaml
로컬_실행:
  1. 기존 프로세스 종료
  2. 새 빌드 실행
  3. 헬스체크 확인

모바일_배포:
  1. 에뮬레이터/디바이스 확인
  2. APK 설치
  3. 앱 실행
  4. 로그 확인

원격_서버_배포:
  1. 배포 스크립트 실행
  2. 상태 확인
  3. 로그 확인
```

## 선택적 배포

> 선택적 빌드(kTest_build)에서 결정된 빌드 대상 프로젝트만 배포. 빌드하지 않은 프로젝트는 배포 스킵.

```yaml
원칙:
  빌드_대상만_배포: 선택적 빌드로 빌드된 프로젝트만 배포
  미빌드_프로젝트: 기존 실행 상태 유지 (종료/재시작 금지)
  Fallback: 빌드 대상 정보 없으면 전체 배포

배포_스킵_판단:
  조건: 선택적 빌드에서 "빌드 스킵" 판정 (비코드 파일만 변경)
  행동: 배포도 스킵 → kTest_run으로 직행 (기존 실행 상태 유지)

연동_방식:
  kTest_build → 빌드 대상 목록 → kTest_deploy에 전달
  전달_방법: 에이전트 프롬프트에 빌드 대상 명시
```

## 배포 설정

```yaml
프로젝트별: kInfra_{project} 프로젝트스킬 참조
필수: 빌드 성공 후에만 배포
헬스체크: 배포 후 반드시 실행
실패_시: kDev 복귀
```
