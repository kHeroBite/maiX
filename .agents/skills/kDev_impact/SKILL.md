# kDev_impact — 영향도 분석

## 목적

구현 완료 후, 변경된 심볼/인터페이스의 **참조를 자동 추적**하여 누락된 수정 지점을 감지합니다.

## 활성화 조건

```yaml
필수:
  - 인터페이스/메서드 시그니처 변경 포함 시
  - 공통모듈(Shared) 변경 포함 시
  - 크로스 레이어 변경 시 (DB→Backend→Frontend)

불필요:
  - UI 전용 변경 (단일 폼 내부 수정)
  - 문서/설정 파일만 변경
  - 기존 인터페이스 내 구현 변경
```

## 에이전트 유형

```yaml
유형: Explore (읽기 전용, Serena 심볼 탐색 활용)
모델: Sonnet (참조 추적은 분석력 필요)
제약: 파일 수정 절대 금지 — 분석/보고만
```

## 분석 범위

```yaml
심볼_참조_추적:
  도구: Serena find_referencing_symbols
  대상: 시그니처가 변경된 메서드/클래스/인터페이스
  결과: 해당 심볼을 참조하는 모든 코드 위치

쿼리_참조_추적:
  대상: 변경/삭제/이름변경된 쿼리 (Mars.Shared/Queries)
  방법: Grep으로 쿼리명 참조 검색
  결과: 호출부가 새 시그니처에 맞게 수정되었는지 확인

DB_스키마_영향:
  대상: 테이블/컬럼 변경 시
  방법: 해당 테이블을 참조하는 쿼리 추적
  결과: 쿼리와 코드가 새 스키마에 맞게 수정되었는지 확인
```

## 실행 절차

```yaml
1_리더가_Impact_디스패치:
  시점: 모든 구현 에이전트 완료 후 (코드 리뷰와 병렬)
  방법: Task(Explore) 에이전트 생성
  프롬프트: |
    아래 변경사항의 영향도를 분석하라:

    [변경된 심볼/메서드 목록]
    [변경 유형: 시그니처/이름/삭제/추가]

    Serena find_referencing_symbols로 각 심볼의 참조를 추적하고,
    참조하는 코드가 새 시그니처에 맞게 수정되었는지 확인하라.

    결과 파일: {scratchpad}/impact_report.md
    반환: 누락 수 + 요약 5줄 + 파일 경로

2_코드_리뷰와_병렬_실행:
  둘 다 읽기 전용이므로 충돌 없음

3_결과_처리:
  누락_없음: kTest 단계로 진행
  누락_발견: 리더가 추가 수정 → 재빌드 → 재분석 (최대 1회)
```

## 결과 형식

```markdown
# 영향도 분석 보고서

## 판정: ✅ 누락 없음 / ⚠️ 누락 N건 발견

## 변경 심볼별 참조 현황
| 변경 심볼 | 변경 유형 | 참조 수 | 수정 완료 | 미수정 | 상태 |
|----------|----------|---------|----------|--------|------|

## 미수정 참조 상세 (누락 발견 시)
- [파일:라인] — [참조 코드] — [필요한 수정]

## 크로스 레이어 영향
- [영향 경로: Shared → Mars/MarsGW/Mars.Mobile]
```

## 제한사항

```yaml
읽기_전용: 파일 수정 절대 금지 (누락 발견 시 보고만)
scratchpad_필수: 결과를 반환 메시지에 전체 포함 금지
심볼_추적_제한: 변경된 심볼 최대 20개 (초과 시 리더에게 분할 요청)
```
