# kPlan_sim — 계획 시뮬레이션 검증

## 소속 및 실행 시점

```yaml
소속: kPlan의 서브스킬 (kDev의 서브스킬이 아님)
목적: 세워진 계획(TODO)을 가상으로 시뮬레이션하여 엣지케이스/논리 오류 사전 발견
실행_시점: 계획 수립 후, 구현 전
다음_단계: 시뮬레이션 통과 → Review → kDev(구현)

주의:
  - 구현된 코드를 시뮬레이션하는 것이 아니다
  - 구현 완료 후에는 kTest(테스트)로 진행한다
```

## 시뮬레이션 횟수

```yaml
낮음: 3회 (정상 1 + 예외 1 + 엣지 1)
중간: 최소 5회 (정상 2 + 예외 2 + 엣지 1)
높음: 최소 8회 (정상 3 + 예외 3 + 엣지 2)
```

## 3단 구조

각 시뮬레이션은 다음 형식:

```
### 시뮬레이션 #N: [시나리오명]
- **유형**: 정상 / 예외 / 엣지케이스
- **입력**: [구체적 입력값/조건]
- **기대 결과**: [예상되는 동작/출력]
- **실제 추론**: [코드 추적 결과]
- **판정**: ✅ PASS / ❌ FAIL
- **FAIL 사유**: (실패 시만)
```

## 시나리오 카테고리

```yaml
정상_케이스:
  - 일반적인 사용 시나리오
  - 기본 입력값, 정상 흐름
  - Happy path

예외_케이스:
  - null/빈 값 입력
  - DB 연결 실패
  - 네트워크 타임아웃
  - 권한 없음
  - Gateway 모드 / 직접 연결 모드 전환

엣지_케이스:
  - 동시 접근
  - 대용량 데이터
  - 특수문자/한글 입력
  - 경계값 (0, MAX, 빈 리스트)
```

## 실패 시 처리

```yaml
FAIL_발견: → Deep로 복귀 (계획 수정)
전체_PASS: → Review 진행
```
