# AGENTS.md

이 문서는 Codex CLI가 MaiX 프로젝트에서 따라야 할 추가 운영 메모만 제공합니다.
공통 규칙과 상세 절차는 아래 문서를 반드시 참조합니다.

## 필수 참조 문서
- `CLAUDE.md`: 공통 규칙, 언어/인코딩 정책, MCP/스킬 운용 규칙.
- `PROJECT.md`: MaiX 프로젝트 구조, 개발 사이클, 빌드/테스트/배포 절차.
- `ADVANCED.md`: 고급 운영 정책과 예외 처리 가이드.
- `DATABASE.md`: DB 스키마, FK 관계, MySQL 작업 규칙.
- `MCP.md`: MCP 서버 통합 가이드 및 사용 규칙.
- `restapi.md`, `LESSONS.md`: API/운영 기록 참고 문서.
- `.claude/skills/*`: Claude Code와 Codex가 공유하는 스킬 문서.
- `~/.claude/hooks`: Claude Code 공용 Hook 경로(공통 Hook 참조).

## kO 우선 실행 규칙 (강화)
- 모든 사용자 요청은 기본적으로 `kO` 오케스트레이션 흐름을 우선 적용합니다.
- Codex 세션에서 자동 트리거가 보장되지 않으면 `kO`를 명시적으로 호출해 시작합니다.
- 사용자가 `kO` 미사용을 명시한 경우에만 예외를 허용합니다.
- `kO` 세부 절차/분기 기준은 `CLAUDE.md`와 `kO` 스킬 문서를 단일 기준으로 따릅니다.

## Codex CLI 전용 메모
- Codex CLI는 Claude Code처럼 스킬/훅 자동 실행이 항상 보장되지 않으므로 필요한 스킬 문서를 직접 열어 사용합니다.
- 언어/인코딩, 명령 실행, MCP 사용 규칙은 `CLAUDE.md`를 단일 기준으로 따릅니다.
- 빌드/테스트/버전 관리 절차는 `PROJECT.md`의 개발 사이클을 따릅니다.

## 한국어/인코딩 강화 정책
- 모든 커뮤니케이션(진행 보고, 계획, 결과 요약, 질의응답)은 반드시 한국어로 작성합니다.
- 코드/명령어/파일 경로 외 설명 문장은 한국어를 기본으로 하고, 영어는 필요한 용어에만 제한합니다.
- PowerShell 파일 출력/저장은 UTF-8 인코딩을 명시하고, 대량 치환 후 `git diff`로 한글 손상 여부를 즉시 확인합니다.
- 커밋 메시지는 한국어 한 줄 요약을 기본으로 하며, 프로젝트 규칙에 정의된 접두/형식을 준수합니다.
